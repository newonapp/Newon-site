GitHub 수동 업로드 / Pages 루트에 둘 것 (ZIP 말고, 폴더 구조 그대로)
=========================================================

① 맥에서 먼저:  cd Newon  →  npm run publish
② Finder에서 열 폴더:  _publish  (이 안의 것만 GitHub 루트에 올림)

루트 파일:
  index.html
  styles.css
  ox-month.css
  .nojekyll
  CNAME
  logo.png
  ox-month-logo.png
  feature-grid.png
  hero-promo.png
  step-add-habit.png
  step-daily-check.png
  step-stats.png

반드시 같이 있어야 하는 폴더 (하나라도 빠지면 페이지 깨짐):
  ko/     en/     ja/     es/     pt-br/     fr/     de/     hi/     id/
  locales/
  ox-img/
  i18n-img/

각 언어 폴더 안에는 보통: index.html, privacy/index.html, terms/index.html

올리지 않아도 됨 (사이트 표시와 무관):
  *.zip, upload-to-github.sh, UPLOAD.txt, netlify.toml, vercel.json (Pages만 쓸 때)

GitHub Actions로 배포할 때:
  워크플로는 저장소 전체를 받은 뒤  node scripts/publish-site.mjs  로 위와 같은 _publish 를 만들어 Pages에 올립니다.
  웹에서 루트 파일 몇 개만 올리는 것으로는 대체되지 않습니다.
